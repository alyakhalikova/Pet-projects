# Чек-лист Grey-box тестирования по техническому заданию с доступом к базам данных

## SC21-01 – Поле ввода названия магазина
- [ ] При вводе названия магазина, существующего в БД, результаты корректно отображаются на карте.
- [ ] При вводе несуществующего названия — возвращается корректное сообщение об отсутствии результатов.
- [ ] Проверить, что поиск по частичному названию возвращает все релевантные магазины.
- [ ] Убедиться, что результаты поиска сохраняются в историю (проверить соответствующую таблицу/кэш).
- [ ] В истории отображаются только успешные запросы, приведшие к результатам.

## SC21-02 – Поле выбора типа магазина
- [ ] Значения выпадающего списка соответствуют типам, хранящимся в справочной таблице типов магазинов в БД.
- [ ] При выборе типа «Продуктовые магазины» отображаются только магазины с этим типом в базе.
- [ ] Убедиться, что магазины с некорректно указанным типом (ошибки в БД) не отображаются в результатах.
- [ ] При изменении типа в БД (например, переименование) — изменения корректно отображаются в UI.

## SC21-03 – Кнопка «Поиск»
- [ ] Убедиться, что при нажатии на кнопку «Поиск» отправляется запрос к БД с ожидаемыми параметрами.
- [ ] Проверить корректность SQL-запроса (использование индексов, оптимизация по LIMIT/OFFSET).
- [ ] Если возвращается более 100 магазинов — данные отображаются постранично или частично (если предусмотрено).
- [ ] Проверить, что результат поиска соответствует данным, хранящимся в таблице магазинов.

## SC21-04 – Кнопка «Построить маршрут»
- [ ] При выборе магазина с карты — проверяется наличие координат (lat, lon) в БД.
- [ ] Убедиться, что маршрут строится до корректных координат магазина.
- [ ] При выборе типа магазина — строится маршрут до ближайшего магазина с указанным типом (по геолокации).
- [ ] Проверить, что при отсутствии подходящих магазинов маршрут не строится.
- [ ] Убедиться, что при смене состояния кнопки в UI, маршрут не сохраняется, если пользователь отменил построение.

## SC21-05 – Область карты
- [ ] Убедиться, что область карты подгружает только те точки, которые соответствуют координатам магазинов в БД.
- [ ] Проверить, что точки магазинов соответствуют координатам в базе (особенно при масштабировании или прокрутке карты).
- [ ] При обновлении координат магазина в БД — они корректно обновляются при следующем поиске.

## SC21-06 – Общие требования
- [ ] При повторном открытии приложения данные о маршруте подгружаются из локального хранилища/кэша (или таблицы маршрутов, если используется).
- [ ] Проверить поведение отключения точной геопозиции при низком заряде устройства (возможно, через логирование событий или условия логики).
